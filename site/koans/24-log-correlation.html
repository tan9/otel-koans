Weaving the Signals

A log without context is a voice in a crowd.
A log with a trace ID is a voice you can follow.
logs — before correlation no trace context
10:42:01.233  [order-svc]  INFO  Order created: #9281
10:42:01.471  [inventory-svc]  ERROR  Stock check failed: item SKU-440
10:42:01.892  [order-svc]  ERROR  Order #9281 failed: inventory unavailable
You see the error. But which request caused it? Which trace? There is no thread connecting the log to the request that produced it.
logs — after correlation trace context injected
10:42:01.233  [order-svc]  INFO  Order created: #9281  trace_id=a1b2c3d4e5f6  span_id=00f067aa
10:42:01.471  [inventory-svc]  ERROR  Stock check failed: item SKU-440  trace_id=a1b2c3d4e5f6  span_id=bb902b71
10:42:01.892  [order-svc]  ERROR  Order #9281 failed: inventory unavailable  trace_id=f7e8d9c0b1a2  span_id=cc1d3e4f
Now each log carries a trace_id and span_id. The first two logs share trace a1b2c3d4e5f6 — they belong to the same request. You can jump from the error log straight to its trace.

One field — trace_id — turns isolated log lines into a thread you can follow back to the full request.
